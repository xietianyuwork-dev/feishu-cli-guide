一、飞书 CLI 简介

飞书 CLI（命令行工具）是一套面向 AI Agent 和开发者的飞书开放平台命令行工具。通过飞书 CLI，你可以用命令行方式操作飞书的各类功能，包括文档、表格、日历、邮件、任务、会议等。

💡

飞书 CLI 让 AI Agent 真正拥有操作飞书的能力，实现自动化办公、数据同步、内容管理等场景。

三、各模块详细功能

1. lark-shared — 共享基础

描述：飞书 CLI 的共享基础模块，负责应用配置初始化、认证登录、身份切换、权限与 scope 管理。

主要命令：

  • lark-cli config init --new — 初始化应用凭证配置
  • lark-cli auth login --recommend — 用户登录授权
  • lark-cli auth status — 查看当前认证状态
  • lark-cli auth logout — 退出登录

典型场景：

  • 首次使用飞书 CLI 时配置应用凭证
  • 切换用户身份（user/bot）执行不同操作
  • 检查当前登录状态和已授权权限

2. lark-doc — 云文档

描述：飞书云文档的全面管理能力，支持创建、编辑、读取文档内容，支持 XML 和 Markdown 格式。

主要命令：

  • lark-cli docs +create --api-version v2 — 创建文档
  • lark-cli docs +fetch --api-version v2 — 获取文档内容
  • lark-cli docs +update --api-version v2 — 更新文档
  • lark-cli docs +media-insert — 插入图片/文件
  • lark-cli docs +media-download — 下载文档媒体

典型场景：

  • 创建项目计划、会议纪要、技术文档
  • 批量更新文档内容
  • 在文档中插入图片和附件
  • 将本地 Markdown 文件导入为飞书文档

3. lark-sheets — 电子表格

描述：飞书电子表格的创建和操作能力，支持创建表格、管理工作表、读写单元格、追加行数据。

主要命令：

  • lark-cli sheets +create — 创建电子表格
  • lark-cli sheets +get — 读取单元格数据
  • lark-cli sheets +update — 更新单元格数据
  • lark-cli sheets +append — 追加行数据
  • lark-cli sheets +find — 查找内容
  • lark-cli sheets +manage — 管理工作表

典型场景：

  • 创建数据报表和统计表格
  • 批量导入/导出数据
  • 自动化数据录入
  • 数据分析与可视化

4. lark-base — 多维表格

描述：飞书多维表格（Base）的全面管理能力，支持建表、字段管理、记录读写、视图配置等。

主要命令：

  • lark-cli base +create — 创建多维表格
  • lark-cli base +table — 管理数据表
  • lark-cli base +field — 管理字段
  • lark-cli base +record — 读写记录
  • lark-cli base +view — 管理视图
  • lark-cli base +dashboard — 管理仪表盘
  • lark-cli base +form — 管理表单
  • lark-cli base +workflow — 管理工作流

典型场景：

  • 搭建项目管理系统
  • 创建客户信息库
  • 设计问卷调查表单
  • 构建自动化工作流

5. lark-drive — 云空间

描述：飞书云空间的文件和文件夹管理能力，支持上传下载、复制移动、搜索、权限管理。

主要命令：

  • lark-cli drive +upload — 上传文件
  • lark-cli drive +download — 下载文件
  • lark-cli drive +search — 搜索文件
  • lark-cli drive +manage — 管理文件/文件夹

  • lark-cli drive +permission — 管理权限

典型场景：

  • 整理文件夹结构

四、典型使用场景举例

场景 1：自动化会议管理

📅

从创建会议 → 发送邀请 → 会议纪要 → 任务跟踪的完整闭环

流程：

  1. calendar +create — 创建会议日程，邀请参会人
  2. im +send — 发送会议提醒消息
  3. vc +get — 获取会议记录
  4. minutes +ai — 获取 AI 生成的会议总结和待办
  5. task +create — 将待办转为跟踪任务

场景 2：项目文档管理

📝

项目文档的创建、协作、归档全流程

流程：

  1. docs +create — 创建项目计划文档
  2. base +create — 创建项目跟踪多维表格
  3. wiki +node — 将文档归档到知识库
  4. drive +permission — 设置文档权限
  5. im +send — 通知团队成员

场景 3：数据报表自动化

📊

从数据收集到报表生成的自动化流程

流程：

  1. base +record — 从多维表格读取数据
  2. sheets +create — 创建统计报表
  3. sheets +update — 填充计算结果
  4. docs +create — 生成分析报告
  5. mail +send — 发送报表给相关人员

场景 4：每日工作摘要

⏰

每日站会报告自动生成

流程：

  1. workflow standup-report — 生成今日日程和任务摘要
  2. calendar +agenda — 查看今日会议
  3. task +list — 查看未完成任务
  4. im +send — 发送摘要到工作群

场景 5：知识库建设

📚

企业知识库的系统化建设

流程：

  1. wiki +space — 创建知识空间
  2. docs +create — 编写知识文档
  3. whiteboard +update — 绘制流程图和架构图
  4. wiki +node — 组织文档结构
  5. drive +permission — 设置访问权限

五、快速开始

安装

npm install -g @larksuite/clinpx -y skills add https://open.feishu.cn --skill -y

配置

lark-cli config init --newlark-cli auth login --recommend

验证

lark-cli auth status

六、注意事项

  1. 权限管理：不同操作需要不同的 scope 授权，未授权的 scope 会导致操作失败
  2. 身份切换：部分操作支持 --as user 或 --as bot 切换身份
  3. API 版本：文档操作必须使用 --api-version v2
  4. 超时设置：登录授权等操作需要较长的 timeout（≥ 600s）
  5. 安全规则：不要在代码中暴露 appSecret 等敏感信息
